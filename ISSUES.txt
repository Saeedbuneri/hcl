# KNOWN ISSUES & CURRENT STATE

## 1. Native Dependencies Build Errors (`better-sqlite3` / `sqlite3`)
- Error: `gyp ERR! find VS You need to install the latest version of Visual Studio including the "Desktop development with C++" workload.`
- Cause: Node-gyp cannot compile SQLite native bindings for Windows because Visual Studio C++ build tools are missing.
- Solution: Install "Desktop development with C++" workload via Visual Studio Installer.

## 2. File Locking / Permission Errors (Windows ENOTEMPTY / EBUSY / EPERM)
- Error: Modules like `electron` and `prebuild-install` fail to extract or remove directories during `npm install`.
- Cause: Antivirus, Windows file indexing, or "zombie" Node processes are holding file handles, preventing npm from replacing files.
- Solution: 
  1. Restart the computer to clear OS-level file locks.
  2. Open an Administrator Command Prompt.
  3. Navigate to `C:\Users\msbte\Downloads\hcl-app`.
  4. Run `npm cache clean --force`.
  5. Run `npm install`.
